# 📋 Custom Instructions

This folder contains custom instruction files for GitHub Copilot that define specific coding standards, best practices, and guidelines for this workspace.


## Recommended Instructions from awesome-copilot

These are some recommended instruction files curated from the [GitHub awesome-copilot repository](https://github.com/github/awesome-copilot). To use any of these instructions:

1. Visit the awesome-copilot repository to find the instruction
2. Download or copy the desired `*.instructions.md` file
3. Add it to your workspace's `.github/instructions/` folder
4. Copilot will automatically apply it to your workspace

| Instruction | Description |
|---|---|
| [Blazor](https://github.com/github/awesome-copilot/blob/main/instructions/blazor.instructions.md) | Blazor component and application development patterns |
| [.NET Framework](https://github.com/github/awesome-copilot/blob/main/instructions/dotnet-framework.instructions.md) | .NET Framework project structure, C# language conventions, and best practices |
| [Self-explanatory Code Commenting](https://github.com/github/awesome-copilot/blob/main/instructions/self-explanatory-code-commenting.instructions.md) | Guidelines for writing self-explanatory code with minimal comments |

## How Instructions Work

When you create `.instructions.md` files in your workspace, GitHub Copilot automatically discovers and applies them to provide contextual guidance for code generation and suggestions. Instructions help you:

- Enforce consistent code style and formatting
- Maintain architectural patterns throughout the project
- Apply domain-specific knowledge and best practices
- Keep coding standards consistent across the team
